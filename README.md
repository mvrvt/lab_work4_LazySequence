Данный проект представляет собой реализацию полиморфных абстрактных типов данных (АТД) на языке C++ в рамках лабораторной работы №4. Основной упор сделан на работу с ленивыми вычислениями, бесконечными последовательностями, потоками ввода-вывода и конечными автоматами (Finite State Machine).

**Вариант целевой задачи:** №6 «Реализация машины/автомата состояний» (FSM валидатор Email-адресов).

## 🚀 Архитектурные особенности и достижения

Проект спроектирован с учетом жестких ограничений и требований к производительности:

* **Strict C++ (No Lambdas):** В проекте полностью исключено использование `std::function` и лямбда-выражений. Вся маршрутизация логики и предикаты построены на классических указателях на функции (`typedef`), что делает архитектуру прозрачной и предсказуемой.
* **Алгебра трансфинитных ординалов ($\omega_0$):** Реализована математически строгая структура `Ordinal`. Конкатенация конечных и бесконечных последовательностей обрабатывается за $O(1)$ без "зависаний". Например, размер сцепления двух бесконечностей равен $\omega_0 \cdot 2$, а доступ к элементам второй последовательности происходит по смещенному ординальному индексу.
* **AST-Генераторы:** Операции `Map`, `Skip`, `Concat` не копируют данные и не используют `while(true)`. Они строят абстрактное синтаксическое дерево (AST) генераторов, делегируя вызовы "вниз" по цепочке.
* **Скользящее окно (`SlidingCache`):** Для соблюдения требований по памяти (работа с миллионами элементов без `OutOfMemory`), ленивая последовательность использует `SlidingCache`. Память выделяется только под заданный размер окна (например, 1000 элементов). При переполнении окно сдвигается за $O(1)$ относительных индексов.
* **Универсальные потоки (`Stream<T>`):** Реализованы абстракции `ReadOnlyStream` и `WriteOnlyStream`, которые инкапсулируют работу с `Sequence`, `std::string` и большими файлами (`std::ifstream`/`ofstream`). Чтение гигабайтных файлов происходит "на лету" (пословно), без загрузки всего файла в RAM.

## 🧠 Продвинутая гибкость (Ответы на вопросы с защиты)

Архитектура изначально спроектирована для решения нестандартных краевых задач:
1. **Сцепление (чередование) $N$ бесконечностей ($a_0, b_0, c_0, a_1, b_1...$):** Реализовано через `InterleaveGenerator`, который переключается между внутренними генераторами с помощью операций `%` и `/` над конечной частью ординала.
2. **Чтение файла, обновляемого другим процессом (`tail -f`):** Реализовано через абстракцию `FileTailGenerator`. При достижении `EOF` генератор сбрасывает флаг ошибки (`file.clear()`) и ожидает поступления новых байтов, не прерывая выполнение потока.

## 📂 Структура проекта

```text
├── CMakeLists.txt              # Скрипт сборки проекта и скачивания GTest
├── main.cpp                    # Точка входа, FSM и интерактивное меню
├── src/
│   ├── Generator.hpp           # Интерфейсы и AST-деревья генераторов
│   ├── LazySequence.hpp        # Ленивые списки (бесконечные структуры)
│   ├── SlidingCache.hpp        # Скользящее окно для кэширования элементов
│   ├── Stream.hpp              # Потоки (Sequence, String, File)
│   ├── StateMachine.hpp        # Реализация конечного автомата (FSM)
│   ├── MyUtils.hpp             # Ординальная алгебра (Ordinal) и Optional
│   └── lab2_files/             # Базовые коллекции (ArraySequence, LinkedList и др.)
└── tests/                      # Модульные тесты (Google Test)
    ├── ordinal_tests.cpp
    ├── sliding_cache_tests.cpp
    ├── lazy_sequence_tests.cpp
    ├── stream_tests.cpp
    └── file_string_stream_tests.cpp
```

## 🛠 Сборка и запуск

Проект использует систему сборки CMake. Библиотека Google Test скачивается и линкуется автоматически.

1. **Создание папки для сборки:**
   ```bash
   mkdir build && cd build
   ```
   Генерация файлов сборки:
   ```bash 
   cmake ..
   ```
   Сборка проекта (основное приложение + тесты):
   ```bash
   cmake --build .
   ```
   Запуск приложения:
   ```bash
   ./app_main
   ```


## 🧪 Тестирование
Код покрыт индустриальными модульными тестами (70 тестов), проверяющими граничные случаи, переполнения кэша, ординальную математику и исключения стандартной библиотеки.
Запуск тестов глубоко интегрирован в само приложение. Вы можете запустить их прямо из интерактивного меню app_main (пункт 2), либо выполнив скомпилированный таргет напрямую:
```bash
./app_tests
```
Тесты разделены на 5 Test Suites, покрывающих всю логику:
1. OrdinalTest (Арифметика трансфинитных ординалов)
2. SlidingCacheTest (Управление памятью и смещение окна)
3. LazySequenceTest (Ленивые вычисления и AST конкатенация)
4. ReadOnlyStreamTest & WriteOnlyStreamTest (Потоки на базе коллекций)
5. FileStringStreamTest (Потоки строк и файлов)
